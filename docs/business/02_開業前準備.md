## 1. Day 0 是什麼

Day 0 = 正式開始找客戶**之前**的一次性準備日，預計 **1–2 天**做完。目標只有一個：等你進入 `05_第一個月行動計畫.md` 時，**不再需要停下來註冊任何東西**。

原則：**全部先用免費方案**。這一天理論上可以一毛錢都不花（見 §5 預算表）。

## 3. 逐一說明（含註冊要點）

### 3.1 WhatsApp Business（最重要）

**這是什麼**：WhatsApp 的免費商家版 App，比一般版多了商家資料頁、商品目錄、自動問候語、快速回覆。印尼客戶的詢問、成交、付款確認、售後，**全部**發生在這裡。

**註冊步驟**：
1. 準備一個**專用門號**（強烈建議不要用私人號——生意號會被大量陌生人加）。可以是台灣門號；WA 認門號不認國家，印尼客戶照樣加得到你。
2. 手機安裝「WhatsApp Business」App（綠色底、B 字圖示），用該門號註冊。
3. 設定商家資料：名稱 `POS Easy`、類別選「軟體」、簡介用品牌資料卡的印尼文簡介（見 §6）、營業時間、官網連結。
4. 設定「問候語」（新客戶加入自動回覆）與「快速回覆」（`/harga` 帶出價目等——模板在 `06`／`08` 手冊）。

**要點與坑**：
- 顯示 +886（台灣）號碼會讓部分印尼客戶起疑。入門期可接受（免費產品門檻低）；Stage 1 之後，讓**在地代理用印尼號碼**開一個服務號，是更漂亮的解法。
- 5 分鐘內回覆是成交第一因子——手機通知務必打開。
- 印尼與台灣時差：印尼西部時間（WIB，雅加達）＝台灣時間**減 1 小時**。

### 3.2 Facebook 個人號＋粉專

**這是什麼**：印尼的 FB 社團（Grup）聚集了大量 warung 老闆；**社團只能用個人帳號加入**，粉專（Halaman／Page）則是你的品牌門面。兩個都要。

**步驟**：
1. 用你現有的個人號即可（新註冊的空白帳號發文容易被當詐騙／被社團拒絕）。把個人號整理乾淨：真人頭像、封面圖、近期至少幾則正常貼文。
2. 搜尋並申請加入社團：`UMKM Indonesia`、`Komunitas Pedagang`、`Usaha Warung`、`Bisnis Kuliner` 等關鍵字，各加 3–5 個活躍大團。入團問題誠實回答。
3. **加入後先潛水互動 3–7 天**（按讚、真誠留言），再開始發文——空降就發廣告會被踢。
4. 用個人號建立粉專：名稱 `POS Easy`，頭像／簡介照品牌資料卡。

### 3.3 TikTok

**這是什麼**：印尼是全球最大的 TikTok 市場之一，warung 老闆滑 TikTok 的時間比看 FB 還多。我們用它發 30 秒短影片（痛點小劇場、功能實拍、老闆見證——腳本見 `06`）。

**步驟**：
1. 下載 TikTok，用品牌 email 註冊。
2. @帳號名統一（例：`@poseasy.id`——所有平台用同一個，見 §6）。
3. 到「設定 → 帳號 → 切換為商業帳號」，類別選「軟體／科技」。
4. 個人檔案放：logo 頭像、印尼文一句簡介、WA 連結（wa.me 短連結，做法見 `06`）。

### 3.4 Instagram

**這是什麼**：同 Meta 家族，Reels 就是 IG 版的 TikTok。策略很省力：**TikTok 拍好的影片原封搬過來**，一魚兩吃。

**步驟**：註冊 → 帳號名與頭像同品牌資料卡 → 切換為商業帳號 → 與 FB 粉專綁定（粉專設定內可連結）→ 簡介放 WA 連結。

### 3.5 Google Business Profile

**這是什麼**：Google 的免費商家檔案。有人在 Google 搜 `aplikasi kasir gratis`（免費收銀 App）時，你的商家卡片有機會出現。免費、十分鐘設完、長期有效。

**步驟**：
1. 到 business.google.com，用品牌 email 登入，點「新增商家」。
2. 名稱 `POS Easy`，類別選「軟體公司」。
3. **地址問題（誠實說明）**：你在印尼沒有實體地址。選「我提供線上服務／外送服務，不設實體店面」的選項，服務區域填目標城市（例：Jakarta、Surabaya）。日後有代理據點再補實體地址。
4. 放上官網（Landing Page 網址）、WA 號碼、營業時間、幾張產品截圖。

### 3.6 Supabase（你的總開關，最重要的技術帳號）

**這是什麼**：一個雲端資料庫服務。客戶的授權（誰付了錢、開了什麼功能）全部存在這裡的 `licenses` 表；`01` 講過——**商業模式是後台設定**，而後台就是它。誰握有這個帳號，誰就握有整門生意的鑰匙。

**步驟**：
1. 到 supabase.com，用品牌 email 註冊（**用主 email，不要用一次性信箱**）。
2. 立刻開啟兩步驟驗證（2FA）：Account → Security。
3. 專案（Project）由開發端建立或依 `SETUP_SUPABASE.md` 建立；你要確認的是**自己是 Owner** 且登入得進去 Table Editor。
4. **保密鐵律**：密碼與 `service_role` 金鑰絕不給任何人（包括代理）。代理永遠只有「賣」的權利，開通鑰匙在你手上（`01` §4.2）。

**免費額度**：入門期（幾百家店）完全夠用；用量大了 Supabase 會先通知，屆時再升級（約 US$25/月起，那是甜蜜的煩惱）。

### 3.7 Vercel

**這是什麼**：免費的網站架設平台。我們的一頁式官網（Landing Page，痛點→方案→WA 按鈕）放這裡，連結 GitHub 後每次更新程式會自動重新上線。

**步驟**：到 vercel.com → 用 GitHub 帳號登入 → Import 專案（Landing Page 屬任務 A2，由開發端完成部署；你要做的是持有帳號、確認網址能開）。免費額度對一頁式網站綽綽有餘。

### 3.8 網域（網址）——可延後

**這是什麼**：品牌網址，例如 `poseasy.com`。沒有它時，Vercel 會免費給你 `xxx.vercel.app` 的網址，**入門期完全夠用**。

**要點**：
- `.id` 結尾的印尼網域**需要印尼身分證（KTP）或印尼法人**才能註冊——你目前辦不到，別浪費時間。先用 `.com`（約 US$10–12/年，任何國際註冊商如 Namecheap／Cloudflare 都可買）。
- 什麼時候買：Stage 1 開始收錢、確定要長期做時。網址要印在傳單與收據上，買了就別再換。

## 4. 收款準備：印尼客戶的錢怎麼進你口袋

先講結論：**這是全份清單裡最需要誠實面對的一題。** 印尼客戶習慣的付款方式，幾乎都需要「印尼身分」才能收；你人不在印尼，入門期的答案是**在地代收節點**。

### 4.1 印尼客戶習慣怎麼付錢

| 方式 | 白話說明 | 收款方需要什麼 |
|---|---|---|
| 銀行轉帳（Transfer Bank） | BCA、BRI、Mandiri 等銀行間互轉，最普遍的 B2B 付法 | 印尼銀行帳戶（開戶需 KTP／居留證） |
| QRIS | 全國統一收款 QR code，客人用任何電子錢包掃 | 印尼商家身分註冊 |
| 電子錢包 | GoPay、OVO、DANA 個人間轉帳 | 印尼手機號＋通常需 KTP 完成實名 |

共同點：**都要印尼身分**。信用卡？warung 老闆基本沒有，不用考慮。PayPal？印尼小店幾乎不用，放棄。

### 4.2 你的現實選項（按可行性排序）

1. **在地代收節點（最務實，Stage 1 前必須養出來）**：由你信任的在地人（早期熱情用戶→代理、或在印尼的親友）用他的帳戶收款，定期結算給你，支付 5–10% 代收費。這與 `01` 版本 D 的代理路線天然接軌——代理本來就該管收款。風險控管：金額小、週結、對照客戶總表核帳（`04` §3）。
2. **國際匯款工具（過渡用，限制多）**：Wise 等服務**匯錢進印尼**容易，但**以印尼本地帳戶身分收 IDR** 目前做不到；讓客戶跨國匯款給你則手續費高到不現實（一筆 Rp 49.000 的月費經不起任何跨境手續費）。結論：只適合「代理定期把整筆結算款匯給你」這一段，不適合直接對單店收款。
3. **印尼金流服務商（Stage 2 以後的正規解）**：Xendit、Midtrans、DOKU 這類公司能幫你收 QRIS／轉帳／電子錢包，但**開戶需要印尼法人（PT）**。等生意證明能做，再評估設立公司或找持牌合作方。

### 4.3 入門期的收款劇本（照做）

- **Stage 0（第 1–4 週）**：全免費白老鼠，**不收款**——這題可以先不解。
- **Stage 1（第 2–3 個月）**：開始收錢前，先確定代收節點。若真的還沒有：優先賣**年繳**與**買斷**（筆數少、金額大，攤得起結算成本），並在 WA 談定時就講清楚轉帳對象是誰（`04` SOP 第 1 步）。
- **Stage 2**：代理制上路後，收款全面在地化；並開始評估金流服務商。

## 7. 法務稅務入門提醒（誠實版）

**先講最重要的一句：本手冊不是法律或稅務建議。** 跨境在印尼正式做生意的合規要求，**必須諮詢印尼當地的會計師／律師**。以下只是讓你「聽得懂專業人士在講什麼」的入門名詞：

| 名詞 | 白話解釋 | 跟你的關係 |
|---|---|---|
| PT | 印尼的本地有限公司 | 開金流服務商帳戶、正式開發票都需要法人 |
| PT PMA | 外資可持股的印尼公司 | 外國人正規經營印尼市場的標準路徑，設立與維護成本高 |
| NPWP | 印尼稅籍編號 | 法人與正式報稅需要 |
| PSE | 印尼通訊部的「數位服務提供者」登記 | 對印尼用戶提供 App／雲端服務達一定規模時可能適用 |

**入門期的務實態度**：
- Stage 0–1 是**小規模市場驗證**（幾十家店、月營收數百萬印尼盾），常見做法是先驗證再正規化——但這是風險取捨，不是「不用管」。
- 收入在台灣端記得**誠實申報**（海外／營業所得依你的個人情況，請教台灣的會計師）。
- 到 Stage 2（代理制、金流服務商、白牌）**之前**，把「設印尼法人或找持牌合作方」排進待辦，並找當地專業人士談過一次。
- 對客戶不開空頭支票：入門期收據用簡易 kwitansi（見 `04` §4），不聲稱能開印尼正式稅務發票（faktur pajak）。
